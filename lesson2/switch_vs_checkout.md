# Git Switch vs Git Checkout

## Вступ

У цій лабораторній роботі ви вивчите ключові відмінності між командами `git switch` та `git checkout`. Ви дізнаєтесь, коли і чому використовувати кожну з них, а також розглянете практичні приклади та сценарії використання для оптимізації вашого робочого процесу з Git. ([LabEx][1])

---

## Розуміння `git checkout`

Команда `git checkout` — це базова команда Git, яка використовується для перемикання між гілками або відновлення файлів у робочій директорії. Вона оновлює файли відповідно до вибраної гілки або коміту. ([LabEx][1])

### Приклад перемикання гілки

```bash
cd ~/project
git branch
git checkout feature-branch
git branch
```

Результат покаже, що ви перейшли на гілку `feature-branch`.

---

## Розуміння `git switch`

Команда `git switch` — це новіша команда (з’явилась у Git 2.23), яка спеціально призначена для перемикання між гілками. Вона робить цей процес більш зрозумілим і менш схильним до помилок. ([LabEx][1])

### Приклад

```bash
cd ~/project
git switch main
git branch
```

Тепер активною гілкою стане `main`.

---

## Створення та перемикання гілок з `git switch`

Одна з переваг `git switch` — можливість створити нову гілку та одразу перейти на неї:

```bash
cd ~/project
git switch -c development
git branch
```

Результат: створено нову гілку `development` і ви одразу на ній.

---

## Відновлення файлів за допомогою `git checkout`

Команда `git checkout` також використовується для відновлення файлів.

### Приклад

```bash
echo "Additional content" >> file1.txt
git status
git checkout -- file1.txt
git status
```

Це скасує зміни у файлі `file1.txt` і поверне його до стану поточної гілки. ([LabEx][1])

---

## Перехід до конкретного коміту (`git checkout`)

`git checkout` дозволяє перейти до конкретного коміту, що переводить репозиторій у стан **detached HEAD**.

### Приклад

```bash
git log --oneline
git checkout <commit_hash>
```

Після цього ви можете переглядати стан проєкту в цьому коміті.

Щоб повернутись до гілки:

```bash
git switch development
```

---

## Основні відмінності

- **`git switch`**
  - Використовується для перемикання між гілками
  - Може створювати гілку (`-c`)
  - Простий і зрозумілий у використанні

- **`git checkout`**
  - Універсальна команда
  - Перемикає гілки
  - Відновлює файли
  - Дозволяє переходити до конкретних комітів

Git розділив функціональність, щоб зменшити плутанину:

- `git switch` — для гілок
- `git restore` — для файлів
- `git checkout` — універсальна (старіша) команда ([datacamp.com][2])

## Висновок

У цій лабораторній роботі ви:

- Навчилися перемикатися між гілками за допомогою `git switch` і `git checkout`
- Створювати нові гілки
- Відновлювати файли
- Переглядати конкретні коміти

Рекомендація: використовуйте `git switch` для роботи з гілками, а `git checkout` — для спеціальних випадків (відновлення файлів або перегляд історії). ([LabEx][1])
